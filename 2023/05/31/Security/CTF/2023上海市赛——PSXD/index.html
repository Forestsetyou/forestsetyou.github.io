comlicated http

流量首先向index.php中上传了一个shell.php，接着利用该后门上传敏感数据，寻找与shell.php相关的流量，被加密的流量根据shell.php中的解密函数可以直接解密获得，传入的基本都是执行命令的脚本，将其执行的命令一起打出：

shell.php

本地用于测试的index.php：

流量25000左右会有一个cat flag的命令，那么flag自然就藏在响应中。比较神奇的是响应中的flag加密内容藏在了”line-based text data”中，该内容在TCP追踪流中是不可见的，需要手动选取并复制。

导出字节流可以直接得到有效字符串。直接复制粘贴进burp会有编码转换问题。
导出十六进制流后在010中直接修改16进制字节流也可以，得到的内容直接粘贴进burp即可。
但将有效内容粘贴进vsc中的文本中，会被vsc做编码处理，字符串会被篡改，依然不行。直接导出字节流用vsc打开并复制是可行的。
python中解码十六进制后打印出来，直接复制进burp也不行：

利用burp传入加密数据时，用GET的参数传入会导致URL编码被解码，而从请求的body传入内容不会，后者才能被解码函数接收。

Cookieback

主页下的输入框怎么查询都查不出东西，也没看出Cookie的加密方式有什么可以破解的地方

F12提示中发送data到cookie的意思是访问 /cookie 路由，参数是data，用Get方式可以传递数据。根据提示试着发送Cookie中connect.sid的值，发现没用。带上 connect.sid= 后发送成功，没反应，回退页面自动刷新发现flag：flag{31461e6c-efc2-474b-918e-e242d9bdfea2}
