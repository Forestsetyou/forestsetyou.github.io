Terminal-Conf

配置一个好看、便捷、趁手的终端是很有用的

Windows Terminal

故事要从”Windows Terminal”开始讲起——

好吧不讲故事，Windows 难得干了件人事就是开发了这么一个美观且功能强大的终端集成器（”Windows Terminal”下略为”WT”）

WT 的配置文件一般在 %USERPROFILE%\AppData\Local\Packages\Microsoft.WindowsTerminal_<SomeOrderNumber>\LocalState\ 下，名为”settings.json”，个人参考配置文件放在后面。同个文件夹下还有一个”state.json”文件，不知道如何配置则直接清空，重新启动 WT，会自动配置该文件内容

直接从微软商店下载即可

Powershell Core

Windows 会自带一个 Powershell 叫”Windows Powershell”，其内核版本为 5，而最新版（2023）已经到了 7，在 Github 主页 上下载并安装最新版

为了能让 WT 给 PWSH7 创建动态配置文件，建议将其安装在默认路径下；在”settings.json”中以 "source": "Windows.Terminal.PowershellCore" 类似形式启动的 shell 是来自于动态配置文件的（看起来更优雅.jpg），Windows 会在每次启动 WT 时检查动态配置文件相关的 shell 是否存在，若不存在则修改”settings.json”添加配置文件。

根据微软约定，”Windows Powershell”命名为”powershell.exe”，而更新版本的命名为”pwsh.exe”

pwsh 有自己的配置文件，常使用的配置文件默认为放在 %USERPROFILE%\Documents\WindowsPowerShell\ 下的”profile.ps1”文件。该文件可配置 pwsh 启动前预执行的命令。

pwsh 可能会设置默认策略禁止运行 ps1 脚本（配置文件），管理员模式下使用 set-ExecutionPolicy RemoteSigned 开启脚本的运行权限，来激活配置文件。

Dircolors

Linux 原生的文件树显示命令的文本染色过于阴间，利用 dircolors 命令定制颜色

dircolors -p > ~/.dircolors 将默认颜色配置输出为配置文件”.dircolors”，在文件中配置色彩，随后修改”.zshrc”中的相似命令 dircolors -b ~/.dircolors 指定从该配置文件中读取配置。一般”.zshrc”中会自带该命令，修改文件名即可。

详情参考：dircolors 命令详解
