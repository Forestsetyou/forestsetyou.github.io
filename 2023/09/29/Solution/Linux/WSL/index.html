WSL 1 不需要开启 Hyper-V

如果需要安装 WSL 2，则需要额外的步骤。

首先需要启用系统的 Hyper-V 功能，参考：微软-Win10 启用 Hyper-V

值得一提的是 Windows 系统不论什么版本，在首次安装后默认是启用 Hyper-V 功能的，不需要特地激活，但服务默认关闭。

这意味着你不需要在功能中开启它，而是可以直接管理他的服务，开启服务即打开了系统的 Hyper-V。

当然这并不一定准确，Hyper-V 本身需要”Virtualization Technology”（简称 VT，即虚拟化技术）的支持，如果 CPU 本身就不支持，那 Hyper-V 自然不可能激活。

我自己用的 Windows 家庭版，系统被安装时 Hyper-V 并未启用，我仍可以使用安卓模拟器（例如雷电模拟器）。

安装 Docker Desktop 时，由于默认使用 WSL2 引擎，它自动为我启用了 Hyper-V。

环境适配

除了 WSL，系统环境中一般还有很多其他的虚拟化服务，他们或多或少都对 Hyper-V 的开启或关闭有要求，例如：

  1. Docker，如果使用 Docker Desktop 且使用基于 WSL2 的引擎，则需要开启 Hyper-V。

  2. 虚拟机平台，包括 Vmware、Virtubox 与一众安卓模拟器。他们起初都需要关闭 Hyper-V 才能正常运行。

    但是后来部分软件的引擎得到更新，例如 Vmware16.0 Pro 已经兼容 Hyper-V；安卓模拟器中，国际版蓝叠模拟器（BlueStacks）、微软的”msft-android-emulator”、Mumu 模拟器的”Mumu X”版本等也都支持了 Hyper-V。
