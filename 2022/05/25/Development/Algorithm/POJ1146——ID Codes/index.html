ID Codes

来自：POJ1146

你能相信我因为”No Successor”写成了”NO Successor”而找了一晚上的BUG吗，气死啦

这道题最难的是它的题面英文看不懂，需要借助百度翻译；思路还是很清楚的，就是字典序的定义 ，比较字母与它右侧字母的大小，寻找顺序部分与逆序部分。会观察到字母的变化总是发生在最后一个顺序部分起头的子串，为了方便程序表示，从右往左看，第一次出现 s[i-1]<s[i] 位置i-1就是子串的起点，在剩余部分中找出最小的s[n]使得s[n]>s[i-1]，将s[n]与s[i-1]交换，再将s[i]为起点的子串按字典序全排列即可。

这里有几个技巧，我使用了桶数组统计子串字母的出现情况，便于找出s[n]，也可以直接输出全排列后的子串；在寻找s[n]时还可以在从右往左遍历选出的子串，找出的第一个大于s[i-1]的字母就是最小的，这是因为子串自右往左是非递减的，一开始我为了debug找大佬程序时看到这个片段懵了好久。程序段如下：
